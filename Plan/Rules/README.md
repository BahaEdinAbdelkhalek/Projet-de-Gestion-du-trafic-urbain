# Project Rules

This folder contains rules and guidelines that any AI agent (or developer) must follow when continuing work on this project. These rules ensure consistency across all commits regardless of who is writing the code.

## Files

- coding-style.md - Code formatting, commenting, and documentation rules
- git-workflow.md - Commit messages, branch strategy, and push process
- architecture.md - Project structure, patterns, and conventions used

## Context

- Steps are defined under plan/step-XX-*.md
- The plan is split between you and Baha
- Always follow the exact step file for the current task
